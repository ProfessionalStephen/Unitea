## Context: The Problem

Your Unitea KPI system expects 16 Pipedrive pipelines (defined in `shared/domain/boards.ts`), but your Pipedrive account only has ~2 active pipelines. This causes the Dashboard to display 14 empty/grayed-out boards with no data.

### What We See
- **Expected**: 16 boards with job counts, deal data, health status
- **Actual**: 2 boards with live data; 14 boards with 0 jobs, grayed out
- **Root Cause**: `api/_lib/pipedrive.ts` fetches from Pipedrive API and creates `boardData[pipelineName]` only for pipelines that exist in the account. The frontend `buildPipelineData()` iterates hardcoded BOARDS config (16 items) and fills missing boards with 0s.

### Data Flow
```
1. User clicks "Pull live data" on Dashboard
2. Frontend calls POST /api/pipedrive/pull
3. api/pipedrive/pull calls pullPipedrive(domain, apiKey)
4. pullPipedrive fetches /pipelines endpoint
5. For each pipeline: creates boardData[pipeline.name] with live data
6. Returns: boardData with only 2 entries (because only 2 pipelines exist)
7. Frontend buildPipelineData() iterates BOARDS array (16 items)
8. For each board: checks if boardData[boardName] exists
9. If not: fills with jobCount=0, avgDays=0, status="green"
10. Result: 14 empty boards displayed
```
