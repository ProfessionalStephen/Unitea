## Context: The Problem

Your App.tsx file is 1700+ lines containing 8 major tab features, 20+ helper functions, and 5000+ lines of inline JSX styles. This monolithic structure makes the code hard to navigate, difficult to test individual features, and prone to performance issues. Additionally, critical resolver logic lacks test coverage for edge cases.

### What We See
- **App.tsx**: Single file with all state, all tabs, all UI
- **No component separation**: SetupTab, BoardsTab, TeamTab, etc. all mixed together
- **Styles scattered**: Inline styles everywhere (DARK/LIGHT themes, glass styles, iS, etc.)
- **Test gaps**: 
  - KPI resolver name-case matches untested
  - Time-window aggregates (installs this week, permits, NMA, etc.) untested
  - Edge cases: DST transitions, month boundaries, timezone handling untested

### Why This Matters
- Hard to maintain: 1700 lines = long file to scroll through
- Hard to test: can't test a single tab in isolation
- Hard to optimize: difficult to memoize components without refactoring
- Hard to debug: state changes affect entire file
